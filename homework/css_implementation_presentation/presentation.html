CSS Implementation

How do we apply CSS to an HTML document?

The Three Methods

  • The Good: External Styling
  • The Bad: Internal Styling
  • The Ugly: Inline Styling

The Good Method: External Styling

  • Uses a separate .css file to store all CSS code
  • Uses a link tag within the head tag to link the CSS document to the HTML document

External Styling: Why it’s good

  • Allows the same code to be used in multiple documents
  • Consistency
  • Ease of editing
  • Keeping files separate allows for better organization of web server content
  • Leaves plenty of space to work with (rather than working inside an HTML document)

The Bad Method: Internal Styling

  • Stores all CSS code within the HTML document
  • Code must be within style tags
  • style tags must be within head

Internal Styling: Why it’s bad

  • Clutters up the head element
  • Is used to style an entire document, but is not used in a way that would allow other web pages to use the same style (inconsistency)
  • Lazy

Internal Styling: Why use it at all?

  • Can be used to style an entire webpage differently than other webpages on the same website
  • It’s sorta easier, I guess (if you’re really lazy)

The Ugly Method: Inline Styling

  • HTML elements are styled individually within the document
  • To style an HTML element, the style attribute is used
  • CSS properties and values are stored in quotes as the value for the style attribute

Inline Styling: Why it’s ugly

  • Clutters up individual element tags
  • Difficult to use proper syntax checking/highlighting
  • Only applies to each element on which it’s used (i.e. cannot style one h1 and have the style apply to all the others)
  • Looks gross

Inline Styling: Why use it at all?

  • Can be used to make individual elements appear differently than they otherwise would
  • Sometimes this is what you want, but not often
  • Still better than using old HTML attributes, such as align, border, bgcolor, color
